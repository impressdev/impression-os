# Example — Atlas (multi-page)

The first **multi-page** site generated by Impression OS: a two-page site for
*Atlas*, a fictional knowledge-base company, on the **dark** theme. It exercises
the multi-page pipeline (`plan-site` → `build-site`) and every section type.

## What it demonstrates

- One [brief](brief/brief.json) with two pages → a [site plan](site.json) → a full
  [site build](kit/): **one shared kit** and **two pages** with their own
  templates and metadata.
- The full section library on the landing page: **header → hero → logo-cloud →
  feature-grid → stats → testimonial → pricing → faq → cta → footer** (10
  sections), and an about page: **header → hero → feature-grid → team →
  testimonial → cta → footer** (7 sections).
- The **dark** theme applied site-wide.
- Determinism — the [provenance](provenance.json) checksum matches a fresh build
  (asserted in `tests/site-example.test.mjs`).

## Layout

```
atlas/
├── brief/brief.json     The brief (two pages: / and /about)
├── site.json            The site plan (shared theme + per-page sections)
├── kit/
│   ├── kit.json         The one shared Elementor kit
│   ├── site.json        Page index (path → slug → templates)
│   ├── sitemap.json     Pages (path, slug, title)
│   ├── sitemap.xml      Standard sitemap (path-relative locs)
│   ├── robots.txt       Crawl rules + sitemap pointer
│   ├── site.webmanifest Web app manifest (brand colors from the theme)
│   └── pages/           (page.json: canonical, robots, themeColor, JSON-LD)
│       ├── index/       templates/* + page.json   (landing)
│       └── about/       templates/* + page.json   (about)
└── provenance.json      Checksum over the whole site build
```

## Reproduce it

```bash
node tools/bin/impression.js plan-site examples/atlas/brief/brief.json --out examples/atlas/site.json
node tools/bin/impression.js build-site examples/atlas/site.json --out examples/atlas/kit
node --test tests/site-example.test.mjs
```

The output is byte-identical to what is committed here.

## Import into WordPress + Elementor

Apply `kit/kit.json` once in **Site Settings**, then build each page from the
templates in `kit/pages/<slug>/templates/`, using `page.json` for the page's title
and meta. (Live WordPress verification needs a WordPress environment.)
